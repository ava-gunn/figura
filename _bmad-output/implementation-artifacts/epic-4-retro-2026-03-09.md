## What Went Well

1. **toMiniNotation is textbook TDD** — 14 tests, 100% strudel package coverage, pure string formatting with no music theory logic leaking across the package boundary.

2. **Architecture payoff** — The package boundary held perfectly. `@figura/strudel` imports exactly one type (`ResolvedFigure`) from core and produces plain strings. AGPL isolation is airtight.

3. **Reference phrase end-to-end validation** — The MVP validation gate passes: `1* 3 5 3 2* 1 7- 1*` over `iim7 V7 IM7` resolves to `D4 D4_ A4 B4/2 ~ G4 B3 B3_`. Full pipeline proven from DSL strings to Strudel output.

4. **API surface tests** — Story 4-2 established a valuable pattern: tests that verify the public contract (barrel exports), not just internal behavior.

5. **Code review as quality multiplier** — Reviews caught tie+staccato priority gap, ARCHITECTURE.md velocity typo, missing non-melody figure type test, and tie JSDoc ambiguity. All fixed before story close.

6. **Publint compliance** — Reordered `types` before `import` in both package.json exports fields, preventing silent consumer issues.

## What Didn't Go Well

1. **Tie convention integration bug** — `resolvePhrase` set `tie: true` on the originating event; `toMiniNotation` expected it on the sustained position. Only discovered when Story 4-2 ran the full pipeline end-to-end for the first time. Both sides had passing unit tests — the bug existed in the space between packages.

2. **Pre-existing quality gate issues** — Typecheck, lint, knip, and sherif have been broken since early epics. Every story documents them as "pre-existing, don't fix." Broken gates erode trust in the quality process.

3. **Story spec inaccuracy** — Story 4-2 spec had incorrect chord durations (4/2/2 instead of 3/3/2). The dev caught it, but a wrong spec for a validation story could have produced a passing test validating incorrect output.

4. **Late integration testing** — The full pipeline wasn't exercised end-to-end until the very last story of the very last epic. Earlier integration would have caught the tie convention bug sooner.

## Lessons Learned

1. **Integration bugs hide behind unit test boundaries** — Both sides of the tie convention had passing tests. Cross-package contracts need their own integration tests, introduced early.

2. **Type systems can encode conventions more precisely** — `tie: boolean` was semantically ambiguous about which position carries the flag. Types are better than comments for enforcing conventions.

3. **Quality gate debt compounds silently** — Gates that are always broken train everyone to ignore warnings. Fix them or remove them.

4. **Spec accuracy matters for validation stories** — Story specs defining expected outputs must be cross-checked against source-of-truth docs before development begins.

5. **Architecture investment pays dividends** — Clean package boundaries, pure functions, and typed contracts meant `toMiniNotation` is ~30 lines of trivial string formatting. Complexity lives in the right place.
